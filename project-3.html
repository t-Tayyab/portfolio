QUIZZY

A full-stack quiz platform with user authentication, four knowledge categories, a randomised 5-question engine, real-time feedback, and a MongoDB-backed Express API.

Node.jsExpressMongoDBJavaScriptHTMLCSSREST API

>> PROJECT_OVERVIEW

Quizzy is a full-stack interactive quiz web application built with a JavaScript frontend, a Node.js and Express backend, and a MongoDB database. The application delivers an engaging quiz experience across four knowledge domains — History, Science, Technology, and Sports — with a secure user authentication system, dynamic question delivery, real-time score tracking, and a clean, responsive interface. It was developed and tested on a local environment using localhost.

The Authentication System is fully implemented using Express and MongoDB. New users can sign up with a username and password, with passwords securely hashed before storage. Returning users log in through the same system, and their session is maintained throughout the quiz. This ensures each user's progress and scores are tied to their account, enabling personalised score tracking over multiple sessions.

Once authenticated, users land on the Topic Selection Screen, where they choose one of four quiz categories: History (world events, civilisations, timelines), Science (biology, physics, chemistry, space), Technology (computing, programming, tech history, gadgets), and Sports (world sports, records, tournaments, athletes). Each category contains a curated question bank stored in MongoDB.

The Quiz Engine serves five questions per session, randomly selected from the chosen topic's question bank. Each question is presented as a multiple-choice card with four options. The interface provides immediate visual feedback on answer selection — correct answers highlight in green, incorrect answers reveal both the wrong choice and the correct one. After all five questions, the user is shown a Results Screen with their final score (e.g., 4/5), a performance rating, and the option to retry the same topic or choose a different one.

The Backend API built with Express handles all routing — authentication endpoints (register, login, logout), question fetching (with random selection logic), and score submission. MongoDB stores user accounts and the question bank for all four topics. The separation of frontend and backend through a clean REST API structure makes the application modular and easy to extend with new topics or question sets.

Quizzy demonstrates a complete full-stack architecture — from a hashed password auth system and MongoDB question bank, to a randomised quiz engine and real-time score feedback — all built without a frontend framework, proving end-to-end development competency.

>> CHALLENGES

Implementing secure user authentication from scratch without using a framework like Passport.js required building the full register/login/session flow manually with Express middleware. Ensuring questions were truly random across sessions — without repeating — involved implementing a shuffle algorithm on the MongoDB query results. Keeping the frontend reactive and smooth without a frontend framework (React/Vue) was a deliberate constraint that pushed vanilla JavaScript DOM manipulation skills.

>> OUTCOME

Quizzy is a fully functional full-stack application demonstrating competence in backend development, database design, REST API architecture, and frontend interactivity. The project proves the ability to build secure, data-driven applications end-to-end — from the database layer through the server to the user interface.
